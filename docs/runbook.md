# Operations Runbook: SRE Agent Platform

Living document. Update when something surprises you.

## Local development

### 1. Prerequisites

- Docker Desktop / OrbStack
- Python 3.12, Node 20
- `make`

### 2. Bring up the stack

```
cp .env.example .env
# edit .env: set OPENROUTER_API_KEY at minimum

make dev-up
```

Services:

- Agent API: http://localhost:8000
- OpenAPI docs: http://localhost:8000/docs
- Dashboard: http://localhost:3000
- Chaos UI: http://localhost:8501
- Prometheus: http://localhost:9090

### 3. Run migrations + seed KB

```
cd services/agent
alembic upgrade head
cd ../..
KNOWLEDGE_BASE_ROOT=./knowledge_base python scripts/seed_knowledge_base.py
```

### 4. Trigger a demo scenario

1. Open Chaos UI (http://localhost:8501).
2. Click "Inject 503s on payments-api".
3. Watch the dashboard (http://localhost:3000) - an incident should appear within seconds.
4. If OpenRouter is wired, RCA will populate with supporting evidence from runbooks.

```
export MICROSOFT_APP_TENANT_ID=...

./scripts/deploy-aks.sh
```

## Teams bot setup

1. Create an Azure Bot Service resource (F0 free tier) in Azure portal.
2. Generate an MSA App ID + password (single-tenant recommended).
3. Set the messaging endpoint to `https://<ingress-host>/api/messages`.
4. Add the Teams channel to the Bot Service.
5. Install the bot into your Team via a side-loaded manifest (`teams-app-manifest.json`).
6. Set `MICROSOFT_APP_ID`, `MICROSOFT_APP_PASSWORD`, `MICROSOFT_APP_TENANT_ID` in Helm values.

Adaptive Cards are generated by `infrastructure/messaging/adaptive_cards.py`.

## Common operations

### Seed new runbooks

Drop markdown files in `knowledge_base/runbooks/` with frontmatter `id`, `kind: runbook`,
`title`, and optionally `service`. Re-run `scripts/seed_knowledge_base.py`.

### Rotate OpenRouter key

```
kubectl -n sre-agent create secret generic sre-agent-secrets \
  --from-literal=openrouter_api_key="sk-or-..." \
  --dry-run=client -o yaml | kubectl apply -f -
kubectl -n sre-agent rollout restart deploy/sre-agent
```

### Inspect incident events

```
kubectl -n sre-agent exec -it postgres-0 -- \
  psql -U sre -d sre_agent -c "SELECT event_type, occurred_at FROM incident_events ORDER BY occurred_at DESC LIMIT 20;"
```

### Resume a stuck agent run

The agent pauses at `interrupt_before=["execute"]`. If an approval was acted on outside
the system (e.g. manual), you can resume directly:

```
curl -X POST http://agent:8000/approvals/resolve \
  -H 'Content-Type: application/json' \
  -d '{"approval_id": "appr_...", "decision": "approve", "actor": "you@company.com"}'
```

## Troubleshooting

| Symptom | Likely cause | Fix |
|-----|-----|-----|
| Agent startup fails with "checkpointer setup" | Postgres not reachable | Check `POSTGRES_DSN`, ensure pg pod ready |
| Qdrant search returns 0 results | KB not seeded | Run `seed_knowledge_base.py` |
| Bot posts nothing in Teams | Conversation reference missing | User must DM the bot once to establish reference |
| OpenRouter 429 / 5xx | Rate limit or outage | Circuit breaker opens; agent falls back to low-confidence - verify via `/metrics` |
| Chaos UI can't apply CRDs | RBAC missing | Ensure chaos-ui ServiceAccount has Chaos Mesh role binding |
